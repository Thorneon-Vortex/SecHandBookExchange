# 🚀 前端启动指南

## 📋 前置条件

在启动前端之前，请确保：

1. ✅ **Node.js已安装**
   ```bash
   # 检查Node.js版本（需要16+）
   node --version
   
   # 检查npm版本
   npm --version
   ```

2. ✅ **后端服务已启动**
   - 后端应运行在 `http://localhost:8080`
   - 可以通过访问 `http://localhost:8080/categories` 测试

3. ✅ **数据库已初始化**
   - MySQL数据库已创建
   - 表结构已创建
   - 触发器和存储过程已创建
   - 测试数据已插入

## 🛠️ 安装步骤

### 第一步：进入前端目录

```bash
cd /Users/geniuswan/SecHandBookExchange-master/frontend
```

### 第二步：安装依赖

```bash
npm install
```

> 💡 **提示**: 如果安装速度慢，可以使用国内镜像：
> ```bash
> npm install --registry=https://registry.npmmirror.com
> ```

### 第三步：启动开发服务器

```bash
npm run dev
```

启动成功后，会看到类似输出：

```
  VITE v5.0.0  ready in 500 ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
  ➜  press h to show help
```

### 第四步：访问应用

打开浏览器，访问：`http://localhost:3000`

## 🔧 开发配置

### 修改端口

如果3000端口被占用，可以修改 `vite.config.js`：

```javascript
server: {
  port: 3001,  // 修改为其他端口
  // ...
}
```

### 修改后端地址

如果后端不在localhost:8080，修改 `vite.config.js`：

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://your-backend-url',  // 修改为实际地址
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

## 🐛 常见问题

### 问题1: `npm install` 失败

**解决方案**：
```bash
# 清除缓存
npm cache clean --force

# 使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### 问题2: 端口3000被占用

**解决方案**：
```bash
# 查看占用进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或修改配置文件使用其他端口
```

### 问题3: 请求后端API失败（404）

**原因**: 后端服务未启动或端口不对

**解决方案**：
1. 确保后端在 `localhost:8080` 运行
2. 测试后端：`curl http://localhost:8080/categories`
3. 检查浏览器控制台的Network标签

### 问题4: 登录后显示401错误

**原因**: Token认证失败

**解决方案**：
1. 检查后端的JWT配置
2. 清除浏览器localStorage
3. 重新登录

### 问题5: 页面空白或报错

**解决方案**：
1. 打开浏览器控制台查看错误
2. 检查是否所有依赖都安装成功
3. 尝试删除 `node_modules` 重新安装：
```bash
rm -rf node_modules package-lock.json
npm install
```
